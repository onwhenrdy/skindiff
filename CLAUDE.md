# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`skindiff` is a single R package: a Crank-Nicolson finite-difference
simulator for drug diffusion through a stack of skin compartments
(vehicle / stratum corneum / deeper skin layers / sink), written in C++17
and exposed via Rcpp. There is **no standalone CLI** — the package is
the deliverable.

## Build and run

```sh
# install
"/c/Program Files/R/R-4.5.2/bin/Rscript" -e 'Rcpp::compileAttributes(".")'
"/c/Program Files/R/R-4.5.2/bin/R" CMD INSTALL --no-multiarch .

# build a tarball + R CMD check (must be Status: OK)
cd /tmp
"/c/Program Files/R/R-4.5.2/bin/R" CMD build /c/github/skindiff
"/c/Program Files/R/R-4.5.2/bin/R" CMD check --no-manual skindiff_*.tar.gz

# run all tests (R + C++)
"/c/Program Files/R/R-4.5.2/bin/Rscript" -e 'testthat::test_local("/c/github/skindiff")'
```

If you edit `// [[Rcpp::export(...)]]` annotations in `src/`, rerun
`Rcpp::compileAttributes(".")` to regenerate `src/RcppExports.cpp` and
`R/RcppExports.R` before reinstalling.

If you edit roxygen comments above exported R functions, rerun
`roxygen2::roxygenise(".")` to regenerate `man/*.Rd` and `NAMESPACE`.

## Architecture: data flow

```
R list  ──►  rcpp_bindings::parametersFromR  ──►  Parameters (struct pack)
                                                        │
                                                  System(parameters)
                                                  ├─ builds Geometry
                                                  │  (EQUI_DIST or B_AND_K mesh)
                                                  ├─ builds vector<Compartment>
                                                  │  (vehicle + skin layers)
                                                  ├─ builds Sink
                                                  │  (Perfect_Sink or PK_Compartment)
                                                  ├─ MatrixBuilder::buildMatrix
                                                  │  (Crank-Nicolson, DSkin_1_4)
                                                  └─ initLoggers (MassSeries, CdpSeries)
                                                        │
                                                  System::run()
                                                  for t in 1..sim_time:
                                                    for k in 1..n_ts:
                                                      rhs.inlineMultiply(c)
                                                      thomasReUseIP(lhs, c)
                                                    optionally replace/remove top compartment
                                                    recordAt(t)
                                                        │
                                                  back to R via rcpp_bindings:
                                                    massSeriesToList, cdpToList,
                                                    geometryToList
```

Key points:

- **`Parameters` is a struct pack**, not a class with getters/setters.
  Direct field access throughout (`p.vehicle.D`, not `p.vehicleParameter().D()`).
- **The simulation core is unaware of R**. `System` exposes virtual hooks
  (`progressCallback`, `testForStop`, `initRun`, `tearDownRun`) that
  the R-side `SystemR` (in `rcpp_bindings.cpp`) overrides to forward
  progress and call `Rcpp::checkUserInterrupt()`.
- **Loggers are plain data structs** (`MassSeries`, `CdpSeries` in
  `logger.h`). Mass integration lives in `System::recordAt`. There is
  **no file I/O** — results return to R as in-memory lists / matrices.
- **Only `MatrixBuilder::Method::DSkin_1_4`** is implemented. The older
  `DSkin_1_3` and the `DSkin_1_5` "fast variant" were dropped during
  modernization; if either is needed again, restore from git history.
- **Only Thomas-style solvers** are used in production
  (`thomasIP`, `thomasReUseIP`). The Gauss variants and the
  super-upper / pivot-index members of `TDMatrix` were removed.
- **No JSON parsing**: R lists go straight to `Parameters` via
  `parametersFromR` in `rcpp_bindings.cpp`. The vendored `nlohmann/json`
  is gone.

## R interface contract

Two exported functions:

- `skin_params(vehicle, layers, sink, pk, sim_time, ...)` -- builds and
  validates a parameter list (class `skin_params`). Layers can be a
  data.frame (preferred) or a list of lists. R-side validators check
  types and ranges per field; the C++ `validate()` does cross-cutting
  checks. Both run before the simulation.
- `skin_simulate(params, show_progress = FALSE)` -- runs and returns a
  `skin_result` with `status`, `scaling`, `mass`, `concentration`,
  `cdp`, `geometry`, `params`, `runtime_s`. Print/summary methods are
  defined for both classes.

The `concentration` data.frame is derived in R as `mass / volume` per
compartment (donor volume = `app_area * height`, sink volume = `Vd`).
This is what gives the user "donor concentration over time".

## Testing model

- **C++ tests** live in `src/test-*.cpp` and use testthat's Catch wrapper
  (header-only, comes via `LinkingTo: testthat`). Only `test-runner.cpp`
  defines `TESTTHAT_TEST_RUNNER` so the runner function is emitted
  exactly once. That file also exports a thin Rcpp wrapper
  `.cpp_run_tests()` because `R_useDynamicSymbols(FALSE)` (set by
  `Rcpp::compileAttributes`) hides the raw `run_testthat_tests` symbol
  from R.
- **R tests** use testthat 3.x (`Config/testthat/edition: 3`).

## Header-order trap to remember

In `test-runner.cpp`, **`Rcpp.h` must be included before `testthat.h`**.
`testthat.h` pulls in `Rinternals.h`, which defines `isNull`, `length`
etc. as macros; if Rcpp.h comes after, those macros eat Rcpp's member
functions of the same name and the build fails with a confusing
`Nullable<T>::Rf_isNull(...)` overload resolution error.

## Coordinate / unit conventions in the engine

- `Compartment::height_um` is the physical layer thickness in micrometres.
- `Compartment::area_um2` is `app_area_cm2 * 1e8 * cross_section`.
- `Compartment::c_init` is in `mg / um^3` internally
  (R-side `c_init` is `mg/ml`; `system.cpp` does the `1e-12` conversion).
- Time inside the engine is minutes throughout. PK `t_half` is converted
  from hours to minutes once when constructing the `Sink`.
- `Scaling` (mg/ug/ng) only affects the values stored in `MassSeries` /
  `CdpSeries` (multiplicative factor in `system.cpp`).

## Things to know if you want to undo a simplification

- The old per-method `DSkin_1_3` / `DSkin_1_5` matrix builders are at
  commit `f39d43c~1`.
- The old getter/setter parameter classes are at the same place if you
  ever need the validation patterns they encoded.
- gzipped log file output (`zstr.h`) was dropped — if it comes back, it
  belongs in R, not C++.
