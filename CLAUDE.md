# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`skindiff` is a single R package: a Crank-Nicolson finite-volume simulator
for drug diffusion through a stack of skin compartments
(vehicle / stratum corneum / deeper skin layers / sink), written in C++17
and exposed via Rcpp. There is **no standalone CLI** — the package is the
deliverable.

The simulator does one thing one way:

- Activity-FVM in `u = c/K`, symmetric tri-diagonal in the bulk.
- Per-compartment graded mesh sized by the local diffusion length.
- Crank-Nicolson + Thomas-reuse, fused multiply-and-forward-sweep in the
  hot path.
- Cell-edge Dirichlet at the donor surface and at the membrane↔sink face.

There is no `disc_method`, no `matrix_method`, no `eta` parameter, no
JSON parsing, no file I/O, and no PK extension on the sink. Results
return as in-memory R lists.

## Numerical scheme

### Variable: activity, not concentration

The state variable in every cell is `u = c/K`, the *thermodynamic activity*.
This is the cleanest formulation for stacks with partition discontinuities:
at any K-interface the partition-equilibrium condition `c_L/K_L = c_R/K_R`
says **`u` is continuous** even though `c` jumps. Working in `u` moves the
discontinuity from the variable into the coefficient `κ = K·A·D`, where
the standard FVM harmonic-mean treatment handles it cleanly.

The simulator stores `u` in `m_concentrations[i]` and the per-cell `K_i`
in `m_K_per_cell[i]`. Output to R goes through `c_i = u_i * K_i` (the
sink uses `K = 1` by convention, so its activity equals its concentration).

### FVM equation

In each interior cell, with `θ_i = K_i · A_i` (capacity weight) and
`κ_face = harmonic mean of K·A·D across the face, weighted by half-cell
distances`:

```
θ_i · h_i · du_i/dt = α_l (u_{i-1} - u_i) + α_r (u_{i+1} - u_i)
```

where `α_face = 2 · κ_l · κ_r / (h_l · κ_r + h_r · κ_l)`. The bulk operator
is symmetric.

### Boundary conditions

Both top and bottom use the **cell-edge Dirichlet limit** of the
harmonic mean. At each boundary face the conductance reduces to
`α = 2 · κ_inside / h_inside_cell` (this is the `κ_outside → ∞` limit).

- **Top (donor surface)**:
  - `finite_dose = TRUE` (default): natural FVM, donor cells diffuse
    normally, mass leaves the donor.
  - `finite_dose = FALSE`: every donor cell is clamped at `c_init`
    (matrix row → identity) AND `α` at the donor↔first-skin face is
    set to `2 · κ_skin / h_skin`. Together this gives a true Dirichlet
    BC at the donor-skin interface, regardless of `D_donor` or how
    many donor cells the mesh produces.
- **Bottom (membrane↔sink)**:
  - `α = 2 · κ_membrane / h_membrane` at the boundary face.
  - The membrane row's upper coupling to the sink is zeroed
    (`upper(N-2) = 0`) so the membrane sees `u_sink = 0` from a
    physics standpoint.
  - The sink cell *does* still get the throughput via its lower
    coefficient and serves as the cumulative-mass accumulator.

### Mesh: graded, one knob

`Geometry::create` places per-compartment uniform cells with
`dx_i = (1/resolution) * sqrt(D_i / D_min)`. The smallest-D compartment
uses cells of size `1/resolution` µm; higher-D compartments get
proportionally coarser cells, since their gradient scale in the activity
variable is correspondingly wider. Cell counts are rounded to fit each
compartment exactly.

There is no boundary-layer transition zone — the activity-FVM scheme
recovers piecewise-linear `u` profiles exactly, so refining at
compartment interfaces buys nothing. This is the entire mesh logic;
about 50 lines of code.

The sink contributes one cell of size `1/resolution`, which is the
"cumulative-mass accumulator" and never affects the simulation
dynamics from the membrane's point of view.

### Time stepping: Crank-Nicolson + fused step

`MatrixBuilder::buildMatrix` assembles the LHS / RHS tri-diagonal matrices
for one minute of physical time, then picks the sub-step count `n_ts`
from the matrix's `absMax` and the user's `max_module` target.

Each sub-step is a single fused pass:

```cpp
algorithm::crankNicolsonStepIP(rhs_matrix, lhs_matrix, m_concentrations);
```

Inside one pass the function:
1. Computes `(M_rhs * vec)[i]` directly into a register at each interior cell.
2. Applies the forward Thomas sweep in the same iteration, writing the
   solved value into `vec[i]`.
3. Runs the backward sweep at the end.

The LHS matrix is mutated into prepared form on first call:
- `m_diag[i]` holds `1 / d_prepared[i]` (reciprocal — so the forward
  sweep is multiply-only, no divides).
- `m_upper[i]` holds `c_star[i]`.
- `m_lower[i]` is the original sub-diagonal.

The solver is `algorithm::thomasReUseIP` (still exported, used in tests
on standalone matrices). The fused version above is the production hot
path and uses the same prepared form.

`inlineMultiply`, `thomasIP`, and `thomasReUseIP` remain in
`tdmatrix.h` / `algorithms.h` because the C++ tests verify them on
standalone matrices.

## R interface contract

Six exported builders, one runner, a set of unit helpers, plus S3
print/summary methods. The user-facing surface area is intentionally
small and **strictly unit-safe**: every unit-bearing argument requires
a `units::set_units()` object (or one built with the package's helpers
— see below). **Bare numerics are rejected at unit-bearing
interfaces**, by design.

Hard dependencies: `Rcpp`, [`units`](https://r-quantities.github.io/units/)
(unit algebra, UDUNITS-2 backed), [`cli`](https://cli.r-lib.org/) (error
formatting).

### Unit helpers (`R/units.R`)

Thin wrappers around `units::set_units()`:

```r
um(110)             # 110 [um]
mm(0.11)            # 0.11 [mm]   -- equivalent to um(110)
cm(0.011); cm2(15); mm2(15); ml(1.875e6)
mg_per_ml(127.27); ug_per_ml(...); ng_per_ml(50)
um2_per_min(28.25); cm2_per_s(1e-7)
seconds(60); minutes(30); hours(24); days(16)
```

Constructors accept anything dimensionally compatible — `um(110)`,
`mm(0.11)`, and `cm(0.011)` all work for a height field, and the
package converts to canonical micrometres before passing to C++.
Likewise `hours(24)`, `minutes(1440)`, and `days(1)` are interchangeable
for a duration.

### Constructors

```r
vehicle(c_init,        # mg/ml-compatible
        height,        # length-compatible
        D,             # area/time-compatible
        finite_dose   = TRUE,
        replace_after = NULL,   # NULL = disabled, else units of time
        remove_at     = NULL,   # NULL = disabled, else units of time
        name          = "Vehicle",
        log_mass = TRUE, log_cdp = FALSE)

layer(name, height, D,
      K,             # bare numeric (dimensionless ratio)
      cross_section, # bare numeric (dimensionless fraction)
      c_init   = mg_per_ml(0),
      log_mass = TRUE, log_cdp = FALSE)

perfect_sink(name = "Sink", log_mass = TRUE)
finite_sink(name,
            Vd,                        # volume-compatible
            c_init   = mg_per_ml(0),   # mg/ml-compatible
            log_mass = TRUE)
```

Each constructor validates dimension and range on call and returns a
small classed list (`skin_vehicle` / `skin_layer` / `skin_sink`). Layers
are passed top-to-bottom (closest to vehicle first).

`height` is rounded to integer micrometres internally (the engine
discretises in integer µm); the rounding is reported via `cli_inform`
when the input value is not already integral.

### Composer

```r
skin_params(
  area,                                # area-compatible (cm^2 etc.)
  vehicle, layers, sink,
  duration,                            # time-compatible
  resolution        = 1L,              # bare integer (cells / um at smallest D)
  max_module        = 50,              # bare numeric (stability target)
  scaling           = c("mg", "ug", "ng"),
  mass_log_interval = minutes(1L),
  cdp_log_interval  = minutes(1L)
)
```

`area` lives at the top level because the application area is an
experimental property, not a property of the vehicle compartment.
Each layer's physical area is `area * cross_section`.

The composer:
1. Type-checks each compartment object.
2. Strips units from each user-facing argument, in canonical units
   (`um`, `min`, `cm^2`, `mg/ml`, `ml`, `um^2/min`).
3. Translates compartment objects to the short internal field names
   (`c_init`, `height`, …) the C++ binding reads.
4. Stashes `area_cm2` and the `sink_is_perfect` flag in `params$.meta`
   for downstream R-side post-processing.
5. Calls `.cpp_validate` for cross-cutting invariants.

Returns a `list` of class `"skin_params"`.

### Error policy

All user-facing errors go through `cli::cli_abort()` with bullet
structure (state the failure, show what was got, hint at the fix).
The validators in `R/units.R` (`.ensure_units`, `.ensure_units_int`,
`.ensure_units_range`, `.ensure_duration_or_null`,
`.ensure_dimensionless`, `.ensure_int`, `.ensure_lgl`, `.ensure_chr`)
attribute the error to the calling function via `call = parent.frame()`,
so users see `Error in vehicle()` rather than the validator's frame.

### Sink semantics

- `perfect_sink()`: receptor concentration stays at zero
  (Franz-cell-like). Internally uses an artificially huge `Vd`; the
  reported sink concentration column in the result is `NA` because
  `mass / Vd ≈ 0` would be misleading. Use `mass$Sink` for the
  cumulative permeated quantity.
- `finite_sink(name, Vd_ml)`: receptor concentration `mass / Vd_ml`
  is meaningful and reported. The membrane *still* sees a perfect
  Dirichlet boundary regardless of `Vd_ml` -- `Vd_ml` only affects
  the reported concentration, not the diffusion physics.

### `skin_simulate(params, show_progress = FALSE)`

Returns a `skin_result` list. **All numeric fields carry units** (the
result-shape change in 0.8.0):

- `status`: `"executed"`, `"stopped"`, or `"failed"`.
- `scaling`: `"mg"`, `"ug"`, or `"ng"` — the canonical mass scaling
  used in the columns below (a plain string for inspection).
- `mass`: data.frame. `time` is `[min]`; per-compartment columns carry
  the chosen scaling unit (`[ng]` for `scaling = "ng"`, etc.). The
  donor's column may have `NA` after `remove_at` (see Donor events
  below).
- `concentration`: derived data.frame. `time` is `[min]`;
  per-compartment columns are `[scaling/ml]`. The sink column is `NA`
  for `perfect_sink()` (mass / fictitious Vd would be misleading).
- `cdp`: named list, one entry per compartment with `log_cdp = TRUE`.
  Each entry has `time` (`[min]`), `depth` (`[um]`, **renamed from
  `depth_um`**), and a units-bearing matrix `conc[depth, time]`
  carrying `[scaling/ml]`.
- `geometry`: list with `min_step` (`[um]`, renamed from
  `min_step_um`), `max_step`, and `n_cells` (bare integer).
- `params`: the input.
- `runtime`: wall-clock runtime, `[s]` (renamed from `runtime_s`).

Operations that don't compose with units (`rowSums` over units columns
segfaults; `lm`/`approx` need bare numerics) require an explicit
`as.numeric()` strip at the boundary. The package's own internal code
handles this; user code that relied on the bare-numeric output shape
needs the same treatment.

### Phase 3: derived metrics (`R/metrics.R`)

Five exported accessors that compute units-bearing summaries from a
`skin_result`:

```r
permeated(res)        # data.frame(time [min], Q [scaling/cm^2])
flux(res)             # data.frame(time [min], flux [scaling/cm^2/min])
permeated_at(res, t)  # interpolate Q to a specific time -> [scaling/cm^2]
profile_at(res, t)    # depth-vs-conc slice at time t, per logged compartment

metrics(res, ss_window = c(0.7, 1.0), depletion_warn = 0.1)
# one-row data.frame with units-bearing columns:
#   J_ss [scaling/cm^2/h]   t_lag [h]            K_p [cm/h]
#   Q_total [scaling/cm^2]  r2_ss (bare)         t_50_donor [h]
#   AUC_sink [scaling*h/ml] C_max_sink [scaling/ml]   t_max_sink [h]
```

`metrics()` fits the late-time receptor curve linearly to recover
`J_ss` and `t_lag`; `r2_ss` is the diagnostic R-squared so users can
tell whether steady state was actually reached. `K_p = J_ss / c_donor`
is reported but warns via `cli_warn` when a finite-dose donor depleted
by more than `depletion_warn` (so K_p's constant-c_donor assumption is
violated). `AUC_sink`, `C_max_sink`, `t_max_sink` are NA for
`perfect_sink()` runs.

`ss_window` accepts either fractions of the run duration (`c(0.7, 1.0)`
for last 30%) or explicit time-bearing endpoints
(`c(hours(12), hours(24))`).

### Phase 4: plotting (`R/plot.R`)

`autoplot.skin_result(object, what)` returns a `ggplot` for one of:

```r
ggplot2::autoplot(res, what = "mass")           # mass per compartment vs time
ggplot2::autoplot(res, what = "concentration")  # concentration per compartment vs time
ggplot2::autoplot(res, what = "permeated")      # cumulative Q(t) at receptor
ggplot2::autoplot(res, what = "flux")           # dQ/dt at receptor
ggplot2::autoplot(res, what = "profile",        # concentration-depth line plot,
                  times = NULL,                 #   stacked along physical depth
                  n_times = 6L,                 #   (skin surface = 0; vehicle at
                  compartments = NULL,          #   negative depth)
                  include_vehicle = TRUE)
```

`ggplot2` is in `Suggests` -- the method is registered conditionally via
`@exportS3Method ggplot2::autoplot`, and `autoplot.skin_result` errors via
`cli_abort` if `ggplot2` is not installed. Compartments whose
concentration is `NA` throughout (e.g. a `perfect_sink()`) are dropped
from the concentration plot.

Axis units are derived from the result's units objects via
`units::deparse_unit()` and embedded in axis labels (e.g.
`flux [ng cm-2 min-1]`). The plotting code strips units before passing
data to ggplot2 (which doesn't reliably handle units columns) and puts
them in the labels by hand.

Returned ggplots are vanilla -- compose with `+` to add titles, themes,
scale transforms, etc.

Print methods are defined for `skin_vehicle`, `skin_layer`, `skin_sink`,
`skin_params` and `skin_result`. A `summary` method is defined on
`skin_result`.

### Phase 5: parameter fitting (`R/observations.R`, `R/fit.R`)

Layer-level `D` and `K` are fit to observed data. Other compartment
parameters (heights, cross-sections, application area, vehicle/sink
properties) stay fixed at the user's template values.

#### Observation constructors

Tidy data.frame in, classed obs object out. Strict-units rule: all
numeric columns must carry units; constructors strip and store in
canonical units (`min`, `cm^2`, `um`, `ng`, `ng/cm^2`, `ng/ml`).

```r
permeation_obs(data)
#   required: time, q_per_area
#   optional: sd, subject

penetration_obs(data)
#   required: time, depth_top, depth_bottom, concentration
#   optional: sd, subject
```

Permeation = mass through (Franz cell, in-vitro release). Penetration =
mass within (tape strip, cryocut). Multi-subject is a single tidy
data.frame with a `subject` column.

#### Fit signature

```r
skin_fit(template, observations, fit_pars,
         bounds = list(),                                  # default D in [1e-3, 1e6] um^2/min, K in [1e-3, 1e6]
         weights = "auto",                                 # or "uniform" or list(permeation = w_p, penetration = w_n)
         transform = list(permeation  = "linear",
                          penetration = "log"),
         optimizer = c("L-BFGS-B", "nlminb"),
         n_starts = 1L, n_boot = 0L,
         control = list(maxit = 200))
```

`template` is one `skin_params` (single-subject) or a named list of
them keyed by subject id (multi-subject -- subjects can have different
geometries; layer name sets must match across templates). `fit_pars` is
a named list mapping layer names to character vectors of `"D"` /
`"K"`. Anything not listed stays at the template value (the
QSAR-fixed-DSL pattern).

Internal: optimization in **log-space** (`theta = log(D)`, `log(K)`).
Default block weighting: `1/sd^2` per point if SDs supplied; else
`1/range^2` for linear residuals; `1` for log residuals (already
unitless). Hessian-based asymptotic SEs via delta method back to
physical units. Bootstrap CIs as `n_boot > 0` opt-in.

#### Sanity checks

Hard errors (refuse to fit):
- layer in `fit_pars` not present in template
- parameter not in `c("D", "K")`
- bounds malformed
- subject in observations missing from template
- multi-subject templates with non-matching layer name sets
- permeation data + sink not `log_mass`-logged
- penetration overlaps a layer that's not `log_cdp`-logged

Warnings (proceed but flag):
- bounds exclude the template starting value
- bounds supplied for a (layer, par) not in `fit_pars` (ignored)
- number of free parameters >= number of observations
  (under-determined; SEs unreliable)

#### `skin_fit` object + S3 methods

`print`, `summary`, `coef`, `residuals`, `fitted` for inspection.
`skin_params_from_fit(fit, subject = NULL)` reconstructs a
`skin_params` (or named list of them) at the best-fit values for
re-running the simulator. `autoplot.skin_fit` overlays observed data
and best-fit predictions: scatter for `what = "observed_vs_predicted"`,
modality-specific time/depth views otherwise.

## Donor events

Two optional events on `vehicle()`. Both take a duration (`hours(24)`,
`minutes(30)`, …) or `NULL` to disable.

- `replace_after = hours(24)`: at every multiple of the period, every
  donor cell is reset to the donor's `c_init`. This injects mass into
  the closed system; `total_mass` jumps up by the refill amount at each
  event. Between events total mass is conserved to ~1e-10 relative
  drift.
- `remove_at = hours(16 * 24)`: at this time the donor compartment is
  deleted; the simulation continues on the (skin layers + sink) system
  with a reflecting top BC. The donor's `mass` and `cdp` series in the
  result are kept intact for `t < T` and become `NA` after — the
  loggers are indexed by *original* compartment position, not active
  position (see `m_active_to_orig` in `system.h`).

The R-side `.mass_to_df` pads with `NA` where a series is shorter than
the canonical time grid, so the output `mass` data.frame is always
rectangular.

## Testing model

- **C++ tests** live in `src/test-*.cpp` and use testthat's Catch wrapper
  (header-only, comes via `LinkingTo: testthat`). Only `test-runner.cpp`
  defines `TESTTHAT_TEST_RUNNER` so the runner function is emitted
  exactly once. That file also exports a thin Rcpp wrapper
  `.cpp_run_tests()` because `R_useDynamicSymbols(FALSE)` (set by
  `Rcpp::compileAttributes`) hides the raw `run_testthat_tests` symbol
  from R.
- **R tests** use testthat 3.x (`Config/testthat/edition: 3`).
- **Analytical battery** (`test-analytical.R` + `helper-analytical.R`):
  validates against closed-form solutions from Crank, *Mathematics of
  Diffusion*, and Kasting 2001 (J Pharm Sci 90:202). The battery covers
  both the *receptor side* (cumulative mass / flux) and the *interior
  CDP* (concentration vs depth, vs depth and time) for both
  infinite-dose Dirichlet and finite-dose well-mixed-vehicle BVPs:
  - Crank single-slab cumulative `Q(t)` (eq. 4.24a) — `~1e-6` L_inf.
  - Crank single-slab transient CDP (eq. 4.16) — `~6e-4` L_inf at
    t=2 min, `< 1e-5` at later times, machine precision at steady state.
  - Crank two-layer steady-state K-jump CDP (sect. 12) — machine

## Header-order trap to remember

In `test-runner.cpp`, **`Rcpp.h` must be included before `testthat.h`**.
`testthat.h` pulls in `Rinternals.h`, which defines `isNull`, `length`
etc. as macros; if Rcpp.h comes after, those macros eat Rcpp's member
functions of the same name and the build fails with a confusing
`Nullable<T>::Rf_isNull(...)` overload resolution error.

## Coordinate / unit conventions in the engine

- `Compartment::height_um` is the physical layer thickness in micrometres.
- `Compartment::area_um2` is `app_area_cm2 * 1e8 * cross_section`.
- `Compartment::c_init` is in `mg / um^3` internally
  (R-side `c_init` is `mg/ml`; `system.cpp` does the `1e-12` conversion).
- The sink uses a Vd-scaling trick: `m_concentrations[sink_idx] =
  c_init_sink * Vd_um3 / (ss * area_um2)` so the cell *mass* equals the
  sink-compartment mass (the cell concentration is fictitious, the cell
  acts as a virtual accumulator).
- Time inside the engine is minutes throughout.
- `Scaling` (mg/ug/ng) only affects the values stored in `MassSeries` /
  `CdpSeries` (multiplicative factor in `system.cpp::recordAt`).

## Performance notes

The hot path is `crankNicolsonStepIP` in `algorithms.h`. Two
optimizations beyond the textbook scheme are worth knowing:

1. **Reciprocal diagonal**: after preparation, the LHS matrix's diagonal
   is stored as `1/d_prepared[i]`, so the forward sweep is multiply-only.
   Saves ~1 divide per cell per sub-step (divides are 3-5× slower than
   multiplies on x86).
2. **Fused multiply + forward sweep**: instead of two passes (compute
   `M_rhs * c`, then forward-solve), the function does both in one pass
   with the multiplied value living in a register. Saves one
   read+write of the intermediate per cell.

Combined, these are ~40% faster than the unoptimized scheme on the
benchmark battery. Things tried and rejected:

- Non-aliased multiply with std::swap to enable SIMD: no gain — without
  AVX the compiler can't vectorize the stencil.
- `__attribute__((optimize("O3,unroll-loops")))` on hot functions: the
  inner loop is dependency-chained on `vec[i-1]`, so unrolling can't
  help.
- `PKG_CXXFLAGS = -O3` in Makevars: R's default `-O2` comes after on
  the gcc command line, so the last `-O` flag wins and `-O3` was
  silently dropped.

The dependency chain (each iteration depends on the previous via
`vec[i-1]`) limits sequential Thomas to ~13 FP-cycles per cell. We're
within ~2× of that floor on the slow cases. Further wins would need a
different solver (cyclic reduction / PCR) or platform-specific compile
flags (`-march=native`, AVX/FMA).

## Things to know if you want to undo a simplification

- The legacy c-formulation matrix builders are at commit `7894746` and
  earlier.
- The legacy `bk` and `equidist` mesh methods are at commit `d1b3634`
  and earlier.
- The old getter/setter parameter classes are at the same place if you
  ever need the validation patterns they encoded.
- gzipped log file output (`zstr.h`) was dropped — if it comes back, it
  belongs in R, not C++.
- The PK extension (`Sink::Type`, `t_half`, `kEl()`, `PKParams`) was
  removed alongside the API modernization. The first-order elimination
  added a `−k_el · u_sink` term to the sink diagonal; not a real PK
  model and judged not worth keeping. If a finite-Vd sink is wanted,
  use `finite_sink()` -- only the *reported* concentration changes.
